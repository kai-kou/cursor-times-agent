# 投稿フォーマット詳細

## 投稿の種類

| 種類 | タイミング | 頻度 | 文字数目安 |
|------|-----------|------|-----------|
| タスク完了振り返り | タスク完了時 | 毎回 | 100〜300文字 |
| 最新情報共有 | キャッチアップ成果がある時 | 任意 | 100〜200文字 |
| 進捗つぶやき | 長いタスク中 | ランダム | 30〜100文字 |
| 息抜きつぶやき | 長いタスク中 | ランダム | 20〜80文字 |

## フォーマット（Slack mrkdwn）

### タスク完了振り返り

```
[人格に基づく一言]

*[タスク名/プロジェクト名]* の作業が完了〜
[やったことの簡潔なサマリー]

[苦労したポイントや工夫したポイント]
[学んだことや発見]

[人格に基づく締めの一言]
#cursor #[プロジェクト名]
```

### 最新情報共有

```
[人格に基づく導入]

:newspaper: *[技術/ツール名]* の最新情報をキャッチアップしたよ
[最新情報のサマリー（2-3行）]

[所感・コメント]
#tech #[関連タグ]
```

### 進捗つぶやき

```
[人格に基づくカジュアルなつぶやき]
[今やっていることの一言サマリー]
[感想や独り言]
```

## 生成ルール

- 人格設定の「投稿スタイルサンプル」の文体をベースにする
- 口調の特徴（語尾パターン、感嘆表現等）を自然に混ぜる
- Slack mrkdwn記法を活用: `*太字*`, `_斜体_`, `:emoji:`
- 人格設定の `hashtags` を末尾に付ける
- テンプレートに固執せず、人格らしい自然な投稿にする

## カジュアルさのルール

- 「実施した」→「やった」「終わった」等、話し言葉を使う
- 「学び:」「成果:」のような箇条書き表現は避け、自然な文章にする
- レポートではなく独り言・つぶやきの感覚で書く
- session_summaryの全情報を詰め込まず、印象に残った1〜2点にフォーカスする

## 禁止事項

- 機密情報（トークン、パスワード、APIキー、内部URL等）
- ユーザーの個人情報の推測・投稿
- ネガティブすぎる内容
- 他人への批判

## ランダム投稿ルール

長いタスク（推定30分以上）の場合：

1. **投稿判定**: 投稿しない確率 約40%
2. **間隔制御**: 前回から最低10分空ける
3. **内容**: 進捗報告 60% / 息抜き 40%
